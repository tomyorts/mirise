# 07 Operations Runbook

## 日次運用

### 診療開始前

1. サーバー稼働確認
2. スタッフ端末の充電確認
3. Bluetoothイヤホンの充電確認
4. 主要ルームに接続できるか確認
5. 緊急連絡の代替手段を確認

### 診療中

- 通常連絡は該当ルームで行う
- 全体ルームは緊急または全員周知に限定する
- 患者フルネームや診療内容は極力言わない
- 不要時はPTTを離す

### 診療終了後

1. 端末充電
2. イヤホン清拭・充電
3. 接続トラブルの記録
4. 改善要望の記録

## 障害時対応

### 音が聞こえない

確認順:

1. イヤホン接続
2. スマホ音量
3. アプリの接続状態
4. ルーム選択
5. Wi-Fi接続
6. アプリ再起動
7. 端末再起動

### 全員が使えない

確認順:

1. LiveKitサーバー起動
2. Webアプリ起動
3. Wi-Fiルーター・AP
4. インターネット接続
5. ファイアウォール

### 一部エリアで途切れる

対応:

- 該当場所を記録
- RSSIを測定
- AP配置を確認
- チャンネル干渉を確認
- AP増設を検討

## サーバー操作

### LiveKit起動

```bash
cd infra/livekit
docker compose up -d
```

### LiveKit停止

```bash
cd infra/livekit
docker compose down
```

### ログ確認

```bash
docker compose logs -f livekit
```

## セキュリティ運用

- 退職者は即日アカウント停止
- 紛失端末は即時無効化
- 管理者権限は最小限
- APIキーは共有しない
- 本番環境のシークレットはGitに入れない
