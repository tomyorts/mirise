# 05 Development Roadmap

## Phase 0: 事前準備

目的:

- 技術選定を固定する
- PoC対象院を決める
- 端末とイヤホン候補を決める

成果物:

- 要件定義
- Wi-Fi確認表
- PoC計画
- イヤホン候補リスト

## Phase 1: ブラウザMVP

目的:

- LiveKitで音声インカムが成立するか検証する

実装:

- Next.js Webアプリ
- LiveKit token API
- ルーム選択
- 音声接続
- PTT
- 常時ON
- 緊急全体ルーム切替

合格基準:

- 5台同時接続
- 30分安定
- 受付、診療室、滅菌室で聞き取れる

## Phase 2: 現地PoC

目的:

- 院内Wi-Fiと現場オペレーションで使えるか確認する

検証:

- AP切替
- 音切れ
- Bluetoothイヤホン相性
- バッテリー消費
- スタッフの操作感
- 患者前での違和感

成果物:

- PoCレポート
- 改善一覧
- 採用イヤホン候補
- Wi-Fi改善要件

## Phase 3: React Nativeアプリ化

目的:

- スマホで実用しやすくする

実装:

- iOS/Androidアプリ
- バックグラウンド動作検証
- Bluetoothオーディオルーティング
- イヤホンボタン検証
- 再接続処理
- ログイン
- 端末識別

注意:

OS制約があるため、画面ロック中・バックグラウンド中の動作は実機検証が必須です。

## Phase 4: 管理機能

目的:

- 全院展開できる管理構造にする

実装:

- スタッフ管理
- 施設管理
- ルーム管理
- 権限管理
- 接続ログ
- 端末無効化
- 管理者画面

## Phase 5: 本番運用

目的:

- 1院で本番運用し、全院展開判断を行う

実装:

- HTTPS/WSS
- TURN/TLS
- 監視
- 障害通知
- バックアップ
- 運用マニュアル
- スタッフ研修

## Phase 6: 全院展開

目的:

- ミライズ全体の標準ツールにする

作業:

- 院ごとのWi-Fi調査
- AP改善
- 端末配布
- アカウント発行
- スタッフ説明
- フィードバック回収

## Phase 7: 高度化

候補:

- 複数ルーム待受
- 緊急割り込み
- 音声検知
- ノイズ抑制高度化
- 音声コマンド
- ステータス連携
- 教育用モード
- AIによる業務支援

ただし、録音・文字起こしは最後まで慎重に扱います。
