# 03 Network Wi-Fi Design

## 基本認識

このプロダクトの成否は、アプリよりも院内Wi-Fi品質に強く依存します。
WebRTC音声は低遅延通信のため、電波が弱い、AP間ローミングが不安定、パケットロスが多い環境では音切れします。

## PoC前の最低条件

- 業務用アクセスポイントを使う
- スタッフ専用SSIDを用意する
- ゲストWi-Fiと分離する
- 電子カルテ・レセコン系ネットワークと分離する
- 5GHz優先、対応環境では6GHzも検討する
- AP間ローミングを有効化する
- WMM/QoSを有効化する
- DHCPリースとIP衝突を確認する

## 推奨ネットワーク構成

```text
Internet
  ↓
Firewall / UTM
  ↓
Core Switch
  ├─ Medical System VLAN
  ├─ Staff Intercom VLAN
  ├─ Guest Wi-Fi VLAN
  └─ Admin VLAN
```

## SSID設計

### スタッフインカム用SSID

```text
MIRISE-STAFF-VOICE
```

推奨:

- WPA2/WPA3 Enterpriseまたは強固な共有キー
- ゲストSSIDと分離
- AP間ローミング有効
- 端末数制限を適切に設定

## 電波測定ポイント

院内の以下で必ず測ります。

- 受付
- 待合
- 各診療室
- オペ室
- 滅菌室
- 技工・バックヤード
- スタッフルーム
- 廊下
- 入口付近
- 階段・エレベーター前

## 検証指標

### RSSI目安

- -60 dBm以上: 良好
- -67 dBm以上: 音声用途の目安
- -70 dBm以下: 音切れリスク

### 実測項目

- Ping遅延
- jitter
- パケットロス
- AP切替時の音切れ
- 同時接続台数
- Bluetoothイヤホン使用時の安定性
- 診療時間中の混雑時テスト

## 推奨AP要件

- Wi-Fi 6以上
- 複数AP管理機能
- 802.11k/v/r対応
- VLAN対応
- WMM/QoS対応
- クライアントローミング支援
- 管理画面で端末別RSSI確認可能

## 避けるべき構成

- 家庭用ルーター1台
- ゲストWi-Fiと共用
- AP名だけ同じでローミング制御なし
- 2.4GHz依存
- 電子カルテ系と同一セグメント
- 廊下だけ強く、診療室内が弱い配置

## PoCテスト手順

1. 5台のスマホを用意する
2. 全員が同じルームに接続する
3. 受付、診療室、滅菌室、オペ室へ分散する
4. PTTで順番に発話する
5. 1人が歩きながら発話する
6. AP切替時の途切れを記録する
7. 30分連続でテストする
8. 午前診療または午後診療の混雑時にも実施する

## 合格基準

- 5台接続で30分安定
- 歩行中の音切れが許容範囲
- 診療室から受付への呼び出しが明瞭
- 滅菌室やバックヤードで途切れない
- Bluetoothイヤホン経由でも聞き取れる

## 本番展開前チェック

- 院ごとのAP配置図
- APファームウェアバージョン
- VLAN設計
- ゲストWi-Fi分離
- スタッフ端末管理方針
- 障害時の代替連絡手段
- ネットワーク保守担当の責任範囲
