# 06 Test Plan

## テスト目的

MVPが院内インカムとして実用可能かを、音質、遅延、接続安定性、スタッフ体験の観点から確認します。

## テスト環境

- PoC院: 1院
- 端末: iPhone 3台、Android 2台以上
- イヤホン: 骨伝導、片耳Bluetooth、通常イヤホン
- サーバー: 院内PCまたはMac mini
- ネットワーク: スタッフ専用Wi-Fi

## テスト項目

### 1. 接続テスト

手順:

1. 5台で同じルームに入る
2. 各端末から順番に発話する
3. 接続状態を確認する

合格:

- 全端末で音声が聞こえる
- 接続失敗が少ない

### 2. PTTテスト

手順:

1. ボタンを押して発話
2. 離したらミュート
3. 複数人が連続で発話

合格:

- 押している間だけ声が届く
- 離した後に環境音が漏れない
- 誤操作が少ない

### 3. 常時ONテスト

手順:

1. 常時ONを有効にする
2. 診療室・受付・滅菌室の環境で発話
3. 環境音の入り方を確認

合格:

- 会話が聞き取れる
- 環境音が許容範囲

### 4. 移動テスト

手順:

1. 1人が受付から診療室へ移動
2. 発話しながら廊下を歩く
3. AP切替時の音切れを確認

合格:

- 移動中も大きく途切れない
- 切断しても自動復帰する

### 5. エリアテスト

確認場所:

- 受付
- 待合
- 診療室
- オペ室
- 滅菌室
- バックヤード
- スタッフルーム

合格:

- 主要エリアで実用レベルに聞こえる

### 6. 同時接続テスト

段階:

- 3台
- 5台
- 10台

合格:

- 5台は安定
- 10台で大きな品質低下がない

### 7. バッテリーテスト

手順:

1. スマホ満充電で開始
2. 2時間接続
3. バッテリー消費を記録

合格:

- 半日運用の見込みがある
- 充電運用ルールを作れる

### 8. Bluetooth相性テスト

確認項目:

- 接続安定性
- マイク音質
- 聞き取りやすさ
- 装着感
- 消毒・衛生管理
- 充電持ち
- 片耳/両耳の現場適合性

## 現場ヒアリング項目

- 診療中に操作しやすいか
- 音が聞き取りやすいか
- 患者前で違和感があるか
- 既存連絡方法より便利か
- PTTと常時ONのどちらが使いやすいか
- どのルーム分けが自然か
- 緊急呼び出しは必要か
- 不安な点は何か

## Go / No-Go判断

Go条件:

- 5人同時で30分安定
- 主要エリアで聞き取れる
- スタッフが継続利用に前向き
- Wi-Fi改善が現実的

No-Go条件:

- 音切れが多く業務に使えない
- Bluetooth相性が悪く解決困難
- 操作が現場に合わない
- プライバシー懸念が強い
